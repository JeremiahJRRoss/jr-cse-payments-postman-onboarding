## Slash command index

Each slash command is a markdown file in `.claude/commands/`. The filename
becomes the command (`log-issue.md` → `/log-issue`).

### `/log-issue` — append a structured entry to issues-log.md

Ask the user:
1. What were you trying to do?
2. What was the exact error or unexpected behavior?
3. What did you change?
4. Did AI suggest the fix, or did you find it via docs / `action.yml` / experiment?

Append to `issues-log.md`:

```markdown
### YYYY-MM-DD — <one-line summary>
**Tried:** ...
**Error:** ```<exact text>```
**Changed:** ...
**Source of fix:** AI / Docs / action.yml / experiment / other
```

Append only. Do not commit unless asked.

### `/run-and-watch` — trigger and watch the onboarding workflow

```bash
gh workflow run onboard.yml
sleep 5
RUN_ID=$(gh run list --workflow=onboard.yml --limit 1 --json databaseId --jq '.[0].databaseId')
gh run watch "$RUN_ID"
```

If failed: `gh run view "$RUN_ID" --log-failed`. Summarize the failure and
propose the smallest fix. Do NOT apply without confirmation.

If succeeded: extract outputs (workspace name, spec ID, collection IDs, env IDs,
monitor ID, mock URL) and remind the user to validate in the Postman UI.

### `/validate-postman` — UI validation walkthrough

From the latest green run, extract:
- Workspace name + URL
- Spec ID
- Collection IDs (baseline, smoke, contract)
- Environment IDs
- Monitor ID
- Mock server URL

Present as a numbered list with the URL for each. After each item, wait for
"ok" or an issue. If issue: invoke `/log-issue` flow immediately.

End with: "Ready to re-run for idempotency? (Y/n)"
